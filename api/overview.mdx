## Introduction

The Camera Plugin provides a comprehensive API for accessing device camera functionality in NativePHP Mobile applications. The API is designed to work seamlessly across both PHP (Laravel/Livewire) and JavaScript (Vue/React/Inertia) environments.

## Dual API Architecture

The Camera Plugin offers APIs in two languages:

- **PHP API**: Use `Native\Mobile\Facades\Camera` for server-side operations
- **JavaScript API**: Import from `#nativephp` for client-side interactions

Both APIs provide the same core functionality with language-appropriate syntax and patterns.

## Core Methods

The Camera Plugin exposes three main methods through the bridge system:

### Camera.GetPhoto

Capture a photo using the device camera.

- **PHP**: `Camera::getPhoto()`
- **JavaScript**: `Camera.getPhoto()`
- **Bridge Function**: `Camera.GetPhoto`
- **Android**: `com.nativephp.camera.CameraFunctions.GetPhoto`
- **iOS**: `CameraFunctions.GetPhoto`

### Camera.RecordVideo

Record a video with the device camera.

- **PHP**: `Camera::recordVideo()`
- **JavaScript**: `Camera.recordVideo()`
- **Bridge Function**: `Camera.RecordVideo`
- **Android**: `com.nativephp.camera.CameraFunctions.RecordVideo`
- **iOS**: `CameraFunctions.RecordVideo`

### Camera.PickMedia

Pick media files from the device gallery.

- **PHP**: `Camera::pickImages()`
- **JavaScript**: `Camera.pickImages()`
- **Bridge Function**: `Camera.PickMedia`
- **Android**: `com.nativephp.camera.GalleryFunctions.PickMedia`
- **iOS**: `CameraFunctions.PickMedia`

## Event-Driven Architecture

All camera operations are asynchronous and communicate results through events. The plugin fires events when operations complete, are cancelled, or encounter permission issues.

### Available Events

The following events are registered in the plugin manifest:

- `Native\Mobile\Events\Camera\PhotoTaken`
- `Native\Mobile\Events\Camera\PhotoCancelled`
- `Native\Mobile\Events\Camera\VideoRecorded`
- `Native\Mobile\Events\Camera\VideoCancelled`
- `Native\Mobile\Events\Gallery\MediaSelected`
- `Native\Mobile\Events\Camera\PermissionDenied`

### Event Flow

1. Your application calls a camera method (e.g., `Camera::getPhoto()`)
2. The bridge invokes native code on Android or iOS
3. The native camera interface is presented to the user
4. When the user completes or cancels the action, an event is fired
5. Your event handler receives the result with file paths and metadata

## Bridge Functions

The plugin uses NativePHP's bridge system to communicate between PHP/JavaScript and native code. All three bridge functions are defined in `nativephp.json`:

```json
{
  "name": "Camera.GetPhoto",
  "android": "com.nativephp.camera.CameraFunctions.GetPhoto",
  "ios": "CameraFunctions.GetPhoto",
  "description": "Capture a photo with the device camera"
}
```

Each bridge function:
- Accepts optional parameters (`id`, `event`, etc.)
- Returns an empty map immediately
- Delivers results asynchronously via events

## Fluent API Pattern

Both PHP and JavaScript APIs support a fluent interface for building requests:

```php
Camera::recordVideo()
    ->maxDuration(60)
    ->id('my-recording')
    ->start();
```

```javascript
await Camera.recordVideo()
    .maxDuration(60)
    .id('my-recording');
```
